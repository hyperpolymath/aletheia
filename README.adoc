*Aletheia* is a zero-dependency repository health checker that verifies your project has essential documentation, security disclosure, and build configuration in place.

== What It Checks

Aletheia scans your repository for:

*Documentation*::
README, LICENSE, SECURITY.md, CONTRIBUTING.md, CODE_OF_CONDUCT.md, MAINTAINERS.md, CHANGELOG.md

*Security Disclosure*::
`.well-known/security.txt` (RFC 9116), `.well-known/ai.txt`, `.well-known/humans.txt`

*Build System*::
justfile, flake.nix, CI/CD configuration (.gitlab-ci.yml)

*Project Structure*::
`src/` directory, `tests/` directory

== Quick Start

[source,bash]
----
# Build
cargo build --release

# Check current directory
cargo run

# Check a specific repository
cargo run -- /path/to/repository

# JSON output for CI/CD pipelines
cargo run -- --format json

# Quiet mode (just PASS/FAIL)
cargo run -- --quiet
----

== CLI Usage

[source]
----
aletheia [OPTIONS] [PATH]

OPTIONS:
    -h, --help           Print help information
    -V, --version        Print version information
    -q, --quiet          Quiet mode (only outputs PASS or FAIL)
    -v, --verbose        Verbose mode (detailed output)
    --format <FORMAT>    Output format: human (default), json

EXIT CODES:
    0    All checks passed
    1    Compliance checks failed
    2    Security warning (critical)
    3    Invalid path provided
    4    Invalid arguments
----

== Design Principles

1. *Zero Dependencies* - Only uses Rust standard library (no supply chain risk)
2. *Zero Unsafe Code* - 100% safe Rust
3. *Offline-First* - No network access, works air-gapped
4. *Single Binary* - Easy to distribute and audit (~810 lines)
5. *Self-Verifying* - Aletheia passes its own checks

== Security

- *No unsafe code* - Zero `unsafe` blocks
- *No dependencies* - Eliminates supply chain attack surface
- *No network access* - Cannot exfiltrate data
- *Symlink detection* - Prevents directory traversal attacks
- *Security audit*: 2025-12-26, zero vulnerabilities found
